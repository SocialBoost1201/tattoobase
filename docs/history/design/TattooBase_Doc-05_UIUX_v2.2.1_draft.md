## 1. UI設計思想
TattooBaseは「予約アプリ」ではなく「スタジオ経営OS」であり、**「顧客体験を壊さず、アーティストを裏側で強力に保護する」** ことを至上命題とする。

UIは以下を重視：
- 段階的摩擦設計（優良顧客には1クリック、高リスク顧客にはハードル）
- 意思決定の支援（ビジュアルファーストなArtist検索）
- ネガティブなスコアの不可視化（リスク制御はシステム側で完結）
- 多言語即時切替

## 2. User PWA (顧客側)

### 2.1 グローバル構造
- Header: ロゴ、言語切替、ログイン/マイページ
- Footer: 規約、プライバシー

### 2.2 検索 & 発見体験 (UX強化)
**スタジオ検索から「アーティスト検索」への転換**
- アーティストカードUI：顔写真(アバター)、得意ジャンルタグ、代表作3枚を並べてスワイプ可能に。
- フィルター項目：
  - エリア
  - ジャンル（和彫、洋彫、アニメ、ブラックワーク等）
  - 性別指定（女性アーティスト希望など心理的安全性の確保）
- ポートフォリオ詳細UI:
  - 大きく画像を表示（Pinterest/Instagram風）
  - 画像からそのまま「このテイストで予約相談」へ遷移

### 2.3 予約フロー & デザインデポジット (摩擦最小化)
**前提**: 決済のハードルは高い。必要な場合のみデポジットを要求すること。
1. **メニュー/スタイル選択**
2. **日時選択**（空き枠確認）
3. **ポリシー同意**（シンプルで大きな文字、チェックボックス1つ）
4. **【動的】決済/デポジット要求**
   - **Low Risk (優良顧客) かつ 通常予約**: そのまま予約確定（決済スキップ）
   - **Mid/High Risk または DesignRequest**: 「デザイン作成／枠確保のため、予約金¥〇〇が必要です。ご来店時に施術代金から差し引かれます」という前向きなメッセージと共にStripe Elements表示。
   - ※「あなたは信用度が低いから」といった表現は**絶対に**使用しない。

### 2.4 デザインプレビュー保護UI
**目的**: 盗用・持ち逃げの抑止
- **サムネイル**: 常に低解像度、または強いぼかし。
- **閲覧時**: 
  - 画面全体に「User ID / 日時 / Studio名」の透かし（Watermark）を重ねる。
  - 「現在の閲覧残り回数: N回」を表示。
  - 右クリック・長押し保存禁止のイベントハンドラ（PWA上の気休めだが心理的ハードルにする）。

### 2.5 ドタキャン・遅刻対応UI
- **予約確認画面**: 「遅れそうな場合」「行けなくなった場合」のボタンを大きく配置。
- 「1タップで『5分遅れます』とスタジオに伝える」機能を設け、無断キャンセル（ダメージ最大）を防ぎ、連絡あり遅刻（ダメージ中）に誘導する。

## 3. Studio Web (店舗管理側)

### 3.1 ナビゲーション構造
サイドバー固定：
- ダッシュボード（Today's Action）
- 予約カレンダー / リスト
- 顧客 (CRM)
- アーティスト管理 (新規)
- デザイン管理 (新規)
- 分析
- 設定

### 3.2 アーティスト管理 (負担軽減)
- スタジオオーナーが一括で複数アーティストを登録可能。
- 作品アップロード時、自動的にスタジオの透かしが入る設定項目。
- NSFW（閲覧注意）やジャンルタグの簡単なトグル設定。

### 3.3 デザイン管理 & 例外運用
- DesignRequestのステータス管理ボード（Trello風カンバン）。
- 顧客ごとの進捗：「デポジット待ち」「制作中」「送付済み」。
- **例外コントロール (Audit必須)**: 顧客からの連絡で特例としてデポジットを免除する場合、「免除して予約確定」ボタンを押す。その際、必ず「免除理由（常連、電話で確認済み等）」のテキスト入力モーダルを出し、AuditLogへ流す。

### 3.4 顧客のリスク表示
- CRM画面にて、顧客に内部スコア（数値）は出さず、`[要注意] [優良]` などのバッジアイコンのみを表示。
- キャンセル・遅刻履歴はパッと見で警告色（赤/オレンジ）でハイライト。

## 4. Admin Web (運営側)

### 4.1 メニュー
- 全体ダッシュボード
- KYC承認キュー
- AuditLog監視（例外デポジット免除の監視）

### 4.2 UI制御ルール
- KYC画像は「AI審査NG」のもののみ目視チェック。
- 閲覧時には運営側のAuditLogも残すため、どのAdminがどのKYCを見たか表示・記録。

---
## 5. 多言語UI設計
- 辞書キー必須。未翻訳キーはCI失敗。
- 画面上部のトグルで即時切替（リロードなし）。
- **ユーザー入力（プロフィール、ポートフォリオ説明など）は翻訳不可（AI自動翻訳はPhase2以降）**: 入力された原語のまま表示。
